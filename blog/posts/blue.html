02
Understanding EternalBlue

Most walkthroughs skip this. I won't. Understanding why EternalBlue works is what separates a practitioner from someone who just runs Metasploit.

The vulnerability: MS17-010 exploits a buffer overflow in the Windows SMBv1 server. When Windows processes certain malformed Transaction requests over SMB, an integer overflow allows an attacker to write beyond the bounds of a kernel pool buffer. Because the Windows kernel heap uses predictable pool allocations, a carefully crafted payload can overwrite function pointers in adjacent memory — leading to arbitrary kernel-mode code execution without any authentication.

This was originally developed by the NSA as part of their offensive toolkit, code-named EternalBlue. It was leaked by the Shadow Brokers in April 2017. Within weeks, it was weaponised into WannaCry — which hit 230,000 machines across 150 countries in 24 hours. Microsoft had patched it in March 2017. Blue is running without that patch.

🛡 Blue Team Context

This exploit bypasses all authentication. A firewall rule blocking inbound TCP/445 from external networks prevents remote exploitation — but EternalBlue is often used laterally inside networks, where firewalls don't help. The real fix is patching, disabling SMBv1, and network segmentation.

05
Analysis & Takeaways

Blue isn't technically complex. The lesson here isn't about exploit development — it's about what unpatched infrastructure actually costs. Every organisation running Windows 7 without MS17-010 applied after March 2017 is running this exact scenario. Real attackers know this. Defenders need to know it harder.

Root Cause
SMBv1 not disabled. MS17-010 patch not applied. Windows 7 reached end-of-life in January 2020 — no further patches. Any Windows 7 system still internet-accessible is a ticking clock.
Attack Chain Simplicity
Zero authentication. Zero privilege escalation. One exploit. There was no lateral movement needed. No password cracking. This is what a CVSS 9.3 unauthenticated RCE looks like in practice.
Defensive Fix #1 — Patch
Apply MS17-010. Patch Tuesday exists for a reason. Patch management policy with SLAs for critical vulns (≤30 days) is non-negotiable for any mature security programme.
Defensive Fix #2 — Disable SMBv1
SMBv1 is a 30-year-old protocol with no modern justification. Disable it via Group Policy: Set-SmbServerConfiguration -EnableSMB1Protocol $false. No SMBv1 = EternalBlue doesn't run.
Defensive Fix #3 — Segment Networks
Block inbound TCP/445 and TCP/139 at perimeter and internal segment boundaries. EternalBlue spreads laterally — segmentation limits blast radius even if one machine is compromised.
Defensive Fix #4 — Detect It
IDS/IPS signatures exist for EternalBlue. Suricata rule: ET EXPLOIT MS17-010 EternalBlue. Anomalous SMB transaction patterns on port 445 should trigger alerts and block in your SIEM regardless of patch status.
The Bigger Picture
EternalBlue was patched in 2017. WannaCry hit in May 2017. NotPetya hit in June 2017. Two months. Hundreds of thousands of machines still unpatched. This is not a technical failure — it is an operational security failure. Organisations don't get compromised because they lack talent. They get compromised because patching is unglamorous and unscheduled downtime feels more painful than theoretical risk — until the ransomware hits at 3am on a Friday. This box is a reminder of what that gap costs.
